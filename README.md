# maestro-router

A CLI wrapper that classifies each Claude Code prompt and routes it to the
optimal model + thinking budget. **Works on Claude Pro/Team subscriptions — no
API key required.**

[![License: Apache 2.0](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)

Status: **Pre-release.** v0.0.1-core in progress. Plan: [tasks/todo.md](tasks/todo.md).

## What it does

For every prompt you type in Claude Code, Maestro:

1. Detects turn type (new prompt vs tool result vs error recovery)
2. Classifies complexity (override → turn-type → heuristic, short-circuit at 0.6 confidence)
3. Picks the right `--model`, `--effort`, and `--max-budget-usd` from your profile
4. Adds Claude-specific savings (`--bare` for trivial, `--exclude-dynamic-system-prompt-sections` for cache reuse, per-class tool restriction)
5. Spawns `claude --print --session-id <uuid> --resume ...` so conversation continuity is preserved across model swaps
6. Logs the exact cost + token counts from `--output-format json` to `~/.maestro/decisions.jsonl`

Expected savings vs default Claude Code: 70–80%.

## Quick start

```bash
# 1. Prerequisites (Claude Code installed + authenticated)
claude --version           # need ≥ 2.1.0
claude auth status         # need loggedIn: true (Pro/Team subscription works)

# 2. Install Maestro
npm install -g maestro-router

# 3. Route a single prompt
maestro run "rename foo to bar"             # explicit
echo "design a caching layer" | maestro run # from stdin

# 4. See your savings vs Opus-everywhere
maestro stats
maestro stats --since 30      # last 30 days

# 5. Inspect telemetry
maestro telemetry status
maestro telemetry show --limit 20

# 6. Auto-learn from your override patterns
maestro tune                  # dry-run analysis
maestro tune --apply          # write learned heuristics to ~/.maestro/heuristics.json

# 7. Validate accuracy against the eval set
maestro bench
maestro bench --propose ~/my-overrides.json  # check a proposed change first
```

## Override hints (inline)

Type any of these at the start of a prompt to force a class:

| hint              | class     | model used        |
|-------------------|-----------|-------------------|
| `@fast` `@haiku`  | trivial   | claude-haiku-4-5  |
| `@fast+context`   | trivial   | haiku, full context (no `--bare`) |
| `@sonnet`         | standard  | claude-sonnet-4-6 |
| `@think`          | reasoning | claude-opus-4-7   |
| `@deep` `@opus`   | max       | claude-opus-4-7 @ max effort |

Example: `@deep find the root cause of the flaky test in worker_pool_test.go`.

## VSCode integration

Set `claudeCode.claudeProcessWrapper` in VSCode settings to the path of the
`maestro` binary (`which maestro`). The official Claude Code extension will
launch every Claude invocation through Maestro automatically — both the
panel UI and the integrated terminal route through the wrapper. v0.2.1 will
add a dedicated `maestro --print` wire-compat layer; for now use the `run`
subcommand from the terminal.

## Configuration (all optional)

Three files in `~/.maestro/` let you tune behavior:

```
~/.maestro/config.json              # global preferences (profile, daily cap, etc.)
~/.maestro/profile-overrides.json   # per-class model/effort/budget tweaks
~/.maestro/heuristics.json          # custom regex patterns → classes
```

Generated by `maestro tune --apply`; manually editable.
