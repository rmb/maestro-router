# v0.2.5 Release Plan

## Theme
**"Verify savings recovery + diagnostics"** — ship optimizations from prior spikes, add visibility into cost drivers, gate on eval.

## What We're Shipping

### Merged (7 commits)

#### 1. Session Persistence & Trivial Context (eb43836–7097399)
- **xhigh effort tier**: Assign `effort: "xhigh"` to `reasoning` class (allocates more thinking tokens). Balanced for better debug/incident analysis.
- **No-session-persistence**: Flag `--no-session-persistence` for trivial turns (skips expensive session fingerprinting). Saves ~$0.005/turn on boilerplate edits.
- **trivialMinimalContext**: Strip skills/settings/dynamic sections for trivial prompts. Reduces cache_creation tokens on repeated pattern queries.

**Why:** Optimize the high-volume trivial bucket (68% of turns in typical sessions) without sacrificing quality for complex work.

#### 2. Diagnostics: Cache-Read Visibility (2e5b682)
- Added `cacheReadCostUsd` to telemetry (estimated from `cacheReadInputTokens` × model-specific rates).
- Added `freshSessionRate` counter (% of turns that spawned new sessions).
- Updated `maestro stats` to display:
  - `cache_read cost   $X.XX  (Y% of spend)`
  - `fresh sessions   N%  (M new sessions)`

**Why:** Previous session showed -6% to -21% savings. Root cause analysis (from prior ADR) identified two issues: (1) race condition preventing `lastCacheReadTokens` persistence (fixed in v0.2.4), (2) **zero visibility into whether the fix worked**. Cache-read diagnostics let us prove whether sessions are actually being reused and whether auto-compact is running.

#### 3. Auto-Compact: Telemetry (c1f1472)
- Already shipped in v0.2.4 (af9287e), but added telemetry logging in v0.2.5 (c1f1472).
- Auto-compact now emits a `decision` event with `class: "compact"` when cache_read exceeds threshold.
- `maestro stats` shows `auto_compacts` count.

**Why:** Visibility into when and how often Maestro is auto-compacting. Helps validate that aggressive sessions are actually being managed.

#### 4. Session Race Condition Fix (2503784)
- Shipped in v0.2.4. Included here for completeness.
- Fixed post-turn write race: `stopReason` and `cacheReadInputTokens` both now persist on long sessions.

## What Gets Proven Post-Release

After v0.2.5 ships, monitor telemetry for 3–5 days:

1. **Cache-read costs are positive** (not negative).
   - Check: `maestro stats` shows `cache_read cost $X.XX` where X is small (< 5% of total spend).
   - If cache_read is >10% of spend: auto-compact threshold is too high, or sessions are churning.

2. **Fresh-session rate is low** (<2% on steady-state cwd usage).
   - Check: `maestro stats` shows `fresh sessions <2%`.
   - If >2%: fingerprinting has a bug, or user config is changing frequently.

3. **Auto-compact is firing** (visible in telemetry, not too often).
   - Check: `grep '"class":"compact"' ~/.maestro/decisions.jsonl | wc -l`.
   - If 0: auto-compact logic is never triggered (threshold too high or session caching is broken).
   - If >100 in 1000 turns: threshold too low (wasting compaction overhead).
